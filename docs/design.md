# Interface & Data Design

## Output Format Specification
The output generated by the backend or dashboard must be structured cleanly to ensure output clarity:

| Field | Type | Description |
| :--- | :--- | :--- |
| `candidate_id` | String | Unique candidate identifier |
| `parsed_data` | Object | Skills, experience years, education summary |
| `match_score` | Integer (1-10) | LLM evaluation fit score |
| `justification` | String | Clear explanation of score and candidate strengths/gaps |

## User Workflow Design
1. **User Action**: Upload candidate resume (PDF/Text) and paste Job Description.
2. **Processing**: System parses resume parameters into database format.
3. **Evaluation**: System triggers LLM scoring module to compute candidate fit.
4. **Display**: System presents shortlisted candidates along with structured justifications.
